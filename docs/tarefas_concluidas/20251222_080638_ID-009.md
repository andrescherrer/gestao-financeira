### Características

- ✅ Validação de credenciais (email e senha)
- ✅ Busca de usuário por email
- ✅ Verificação de senha (bcrypt)
- ✅ Verificação de conta ativa
- ✅ Geração de token JWT
- ✅ Retorno de token e dados do usuário

### DTOs

- **LoginInput**: Email, Password
- **LoginOutput**: Token, User (ID, Email, Name)

### Fluxo

1. Valida entrada (email, senha)
2. Busca usuário por email
3. Verifica se usuário existe
4. Verifica se conta está ativa
5. Compara senha com hash
6. Gera token JWT
7. Retorna token e dados do usuário

### Validações

- Email e senha obrigatórios
- Usuário deve existir
- Conta deve estar ativa
- Senha deve ser correta

### Integração

- Usa JWTService para geração de token
- Usa UserRepository para busca
- Usa PasswordHash para verificação

### Testes

- Login com credenciais válidas
- Login com email inválido
- Login com senha incorreta
- Login com conta inativa
- Validação de entrada inválida
