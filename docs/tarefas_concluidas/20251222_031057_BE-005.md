## Resumo da Implementação

**Tarefa:** BE-005 - Implementar endpoints de health check

### Arquivos Criados

- **`backend/pkg/health/health.go`** - Health checker:
  - `LivenessCheck()` - Verifica se a aplicação está viva
  - `ReadinessCheck()` - Verifica se está pronta para receber tráfego
  - Verificação de dependências (database, redis)

### Endpoints Implementados

1. **GET /health/live** - Liveness check
   - Retorna status "alive" e timestamp
   - Sempre retorna 200 OK

2. **GET /health/ready** - Readiness check
   - Verifica conexão com database
   - Verifica conexão com Redis (se configurado)
   - Retorna status detalhado de cada dependência
   - Retorna 200 OK se tudo estiver saudável
   - Retorna 503 Service Unavailable se alguma dependência falhar

### Características

- ✅ Liveness check simples e rápido
- ✅ Readiness check com verificação de dependências
- ✅ Resposta JSON estruturada
- ✅ Timestamps em RFC3339
- ✅ Status HTTP apropriados (200/503)
