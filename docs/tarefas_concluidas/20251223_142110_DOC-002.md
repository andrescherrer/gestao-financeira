**Data:** 2025-12-23

## Resumo da Implementação

**Tarefa:** DOC-002 - Adicionar anotações Swagger nos handlers de Auth

### Anotações Adicionadas/Melhoradas

#### 1. **Register Handler** (`/api/v1/auth/register`)

**Anotações Swagger:**
- @Summary: Register a new user
- @Description: Creates a new user account with email, password, first name and last name
- @Tags: auth
- @Accept: json
- @Produce: json
- @Param: request body (RegisterUserInput)
- @Success: 201 (User registered successfully)
- @Failure: 400 (Bad request - invalid input data)
- @Failure: 409 (Conflict - user already exists)
- @Failure: 500 (Internal server error)
- @Router: /api/v1/auth/register [post]

#### 2. **Login Handler** (`/api/v1/auth/login`)

**Anotações Swagger:**
- @Summary: Login user
- @Description: Authenticates a user with email and password, returns a JWT token for API access
- @Tags: auth
- @Accept: json
- @Produce: json
- @Param: request body (LoginInput)
- @Success: 200 (Login successful)
- @Failure: 400 (Bad request - invalid input data)
- @Failure: 401 (Unauthorized - invalid credentials)
- @Failure: 403 (Forbidden - user account is inactive)
- @Failure: 500 (Internal server error)
- @Router: /api/v1/auth/login [post]

### DTOs Documentados

1. **RegisterUserInput**:
   - email (string, required, email format)
   - password (string, required, min 8 characters)
   - first_name (string, required, min 2 characters)
   - last_name (string, required, min 2 characters)

2. **RegisterUserOutput**:
   - user_id (string)
   - email (string)
   - first_name (string)
   - last_name (string)
   - full_name (string)

3. **LoginInput**:
   - email (string, required, email format)
   - password (string, required)

4. **LoginOutput**:
   - token (string) - JWT token
   - user_id (string)
   - email (string)
   - first_name (string)
   - last_name (string)
   - full_name (string)
   - expires_in (int64) - Token expiration in seconds
