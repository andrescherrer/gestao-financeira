# BUD-003: Criar interface BudgetRepository

**Data:** 2025-12-27 09:43:47  
**Sprint:** 3.1 - Budget Context - Backend  
**Status:** ✅ Completo

## Resumo

Criação da interface `BudgetRepository` no domínio para definir as operações de persistência de orçamentos. A interface pertence à camada de domínio e será implementada na camada de infraestrutura.

## Implementação

### Arquivos Criados

- `backend/internal/budget/domain/repositories/budget_repository.go`

### Métodos Definidos

1. **FindByID**: Busca orçamento por ID
2. **FindByUserID**: Busca todos os orçamentos de um usuário
3. **FindByCategoryAndPeriod**: Busca orçamento por categoria e período
4. **FindByPeriod**: Busca orçamentos por usuário e período
5. **Save**: Salva ou atualiza um orçamento
6. **Delete**: Deleta um orçamento por ID
7. **Exists**: Verifica se um orçamento existe
8. **Count**: Conta orçamentos de um usuário

### Características

- Interface pertence à camada de domínio
- Define contratos de persistência sem dependências de tecnologia
- Permite diferentes implementações (GORM, MongoDB, etc.)
- Segue o padrão Repository do DDD

## Padrão Seguido

Seguido o mesmo padrão usado em `AccountRepository`, `TransactionRepository` e `CategoryRepository`, garantindo consistência na arquitetura.
