# CAT-001: Criar value object CategoryID

**Data:** 2025-12-26 09:41:34  
**Sprint:** 2.3 - Category Context - Backend  
**Status:** ✅ Completo

## Resumo

Criação do value object `CategoryID` para representar identificadores únicos de categorias no domínio. O value object garante que apenas UUIDs válidos sejam aceitos como identificadores de categorias.

## Implementação

### Arquivos Criados

- `backend/internal/category/domain/valueobjects/category_id.go`
- `backend/internal/category/domain/valueobjects/category_id_test.go`

### Funcionalidades

1. **NewCategoryID**: Cria um novo `CategoryID` a partir de uma string, validando se é um UUID válido
2. **GenerateCategoryID**: Gera um novo `CategoryID` com UUID aleatório
3. **MustCategoryID**: Cria um `CategoryID` e faz panic se inválido (útil para testes)
4. **Value**: Retorna o valor do ID como string
5. **String**: Implementa `fmt.Stringer`
6. **Equals**: Compara dois `CategoryID` para igualdade
7. **IsEmpty**: Verifica se o ID está vazio

### Validações

- ID não pode ser vazio
- ID deve ser um UUID válido (formato UUID v4)

### Testes

Testes unitários cobrindo:
- Criação com UUID válido
- Criação com UUID inválido
- Criação com string vazia
- Geração de IDs únicos
- Métodos de comparação e verificação

## Padrão Seguido

Seguido o mesmo padrão usado em `AccountID` e `TransactionID`, garantindo consistência na arquitetura.
