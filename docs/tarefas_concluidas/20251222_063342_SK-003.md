## Resumo da Implementação

**Tarefa:** SK-003 - Implementar value object AccountContext

### Arquivo Criado

- **`backend/internal/shared/domain/valueobjects/account_context.go`** - Value object AccountContext:
  - Representa o contexto de uma conta (Personal ou Business)
  - Validação de valores permitidos
  - Métodos de conveniência para verificação de tipo

### Contextos Suportados

- **PERSONAL** - Conta pessoal
- **BUSINESS** - Conta empresarial

### Características

- ✅ Validação de valores permitidos
- ✅ Métodos de verificação de tipo (IsPersonal, IsBusiness)
- ✅ Nomes de exibição (DisplayName)
- ✅ Funções de conveniência (PersonalContext, BusinessContext)
- ✅ Parser de strings (ParseAccountContext)
- ✅ Lista de todos os contextos (AllAccountContexts)
- ✅ Normalização automática (uppercase, trim)

### Métodos Principais

- **Criação**: `NewAccountContext`, `MustAccountContext`, `ParseAccountContext`
- **Validação**: `IsValidAccountContext`
- **Acesso**: `Value`, `DisplayName`, `String`
- **Verificação**: `IsPersonal`, `IsBusiness`
- **Conveniência**: `PersonalContext`, `BusinessContext`, `AllAccountContexts`
- **Comparação**: `Equals`

### Uso

- Usado na entidade Account para definir o contexto
- Permite separação de contas pessoais e empresariais
- Facilita filtros e consultas por contexto
