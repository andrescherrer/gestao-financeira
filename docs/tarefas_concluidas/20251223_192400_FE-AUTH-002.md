### Fluxo de Registro

1. Usuário preenche todos os campos
2. Validação client-side (Zod)
   - Verifica formato de email
   - Verifica tamanho mínimo de campos
   - Verifica se senhas coincidem
3. Requisição para API (`/auth/register`)
4. Se sucesso:
   - Mensagem de sucesso exibida
   - Redirecionamento automático para `/login` após 2s
5. Se erro:
   - Mensagem de erro exibida
   - Usuário pode tentar novamente

### Schema de Validação

```typescript
const registerSchema = z.object({
  email: z.string().email("Email inválido"),
  password: z.string().min(8, "Senha deve ter no mínimo 8 caracteres"),
  confirmPassword: z.string().min(8, "Confirmação de senha deve ter no mínimo 8 caracteres"),
  firstName: z.string().min(2, "Nome deve ter no mínimo 2 caracteres"),
  lastName: z.string().min(2, "Sobrenome deve ter no mínimo 2 caracteres"),
}).refine((data) => data.password === data.confirmPassword, {
  message: "As senhas não coincidem",
  path: ["confirmPassword"],
});
```

### Diferenciais da Página de Registro

- **Validação de confirmação de senha**: Usa `.refine()` do Zod para validar que as senhas coincidem
- **Grid layout**: Nome e sobrenome lado a lado para melhor UX
- **Feedback de sucesso**: Mensagem verde indicando sucesso antes do redirecionamento
- **Redirecionamento automático**: Após 2 segundos, redireciona para login
