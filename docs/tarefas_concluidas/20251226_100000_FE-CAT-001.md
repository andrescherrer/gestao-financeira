# FE-CAT-001: Criar hook useCategories (TanStack Query)

**Data:** 2025-12-26 10:00:00  
**Sprint:** 2.4 - Módulo de Categorias - Frontend  
**Status:** ✅ Completo

## Resumo

Criação da store Pinia `useCategoriesStore` para gerenciar o estado de categorias no frontend. A store segue o mesmo padrão usado em `useAccountsStore` e `useTransactionsStore`, utilizando Pinia para gerenciamento de estado reativo.

## Implementação

### Arquivos Criados

- `frontend/src/stores/categories.ts`
- `frontend/src/api/categories.ts`
- `frontend/src/api/types.ts` (atualizado com tipos de Category)

### Funcionalidades

1. **Store Pinia (`useCategoriesStore`)**:
   - Estado: `categories`, `currentCategory`, `isLoading`, `error`
   - Computed: `totalCategories`, `activeCategories`, `inactiveCategories`
   - Ações: `listCategories`, `getCategory`, `createCategory`, `updateCategory`, `deleteCategory`, `clearError`, `clearCurrentCategory`

2. **API Service (`categoryService`)**:
   - `list(isActive?)`: Lista categorias com filtro opcional por status
   - `get(categoryId)`: Busca categoria por ID
   - `create(data)`: Cria nova categoria
   - `update(categoryId, data)`: Atualiza categoria existente
   - `delete(categoryId)`: Deleta categoria

### Tipos TypeScript

- `Category`: Interface para categoria
- `CreateCategoryRequest`: Dados para criação
- `UpdateCategoryRequest`: Dados para atualização
- `ListCategoriesResponse`: Resposta da listagem

## Padrão Seguido

Seguido o mesmo padrão usado em `useAccountsStore` e `useTransactionsStore`, garantindo consistência na arquitetura do frontend.
