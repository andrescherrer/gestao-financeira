# CAT-003: Criar interface CategoryRepository

**Data:** 2025-12-26 09:41:34  
**Sprint:** 2.3 - Category Context - Backend  
**Status:** ✅ Completo

## Resumo

Criação da interface `CategoryRepository` no domínio, definindo o contrato para persistência de categorias. A interface pertence à camada de domínio e será implementada na camada de infraestrutura.

## Implementação

### Arquivos Criados

- `backend/internal/category/domain/repositories/category_repository.go`

### Métodos Definidos

1. **FindByID**: Busca uma categoria por ID
2. **FindByUserID**: Busca todas as categorias de um usuário
3. **FindByUserIDAndActive**: Busca categorias de um usuário filtradas por status ativo
4. **Save**: Salva ou atualiza uma categoria
5. **Delete**: Deleta uma categoria (soft delete)
6. **Exists**: Verifica se uma categoria existe
7. **Count**: Conta o total de categorias de um usuário

### Princípios

- Interface no domínio, implementação na infraestrutura
- Retorna entidades de domínio, não modelos de persistência
- Suporta soft delete através de `DeletedAt`
- Filtros por usuário e status ativo

## Padrão Seguido

Seguido o mesmo padrão usado em `AccountRepository` e `TransactionRepository`, garantindo consistência na arquitetura.
