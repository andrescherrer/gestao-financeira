## Resumo

Criação do `CategoryHandler` para lidar com requisições HTTP relacionadas a categorias. O handler implementa todos os endpoints CRUD e integra com os use cases correspondentes.

## Implementação

### Arquivos Criados

- `backend/internal/category/presentation/handlers/category_handler.go`

### Endpoints Implementados

1. **POST /categories** - Create
   - Cria uma nova categoria
   - Requer autenticação
   - Body: `{ name, description }`
   - Retorna 201 com dados da categoria criada

2. **GET /categories** - List
   - Lista todas as categorias do usuário autenticado
   - Query param opcional: `is_active` (true/false)
   - Retorna 200 com lista e contagem

3. **GET /categories/:id** - Get
   - Busca uma categoria específica por ID
   - Retorna 200 com dados da categoria ou 404 se não encontrada

4. **PUT /categories/:id** - Update
   - Atualiza nome e/ou descrição
   - Body: `{ name?, description? }` (pelo menos um obrigatório)
   - Retorna 200 com dados atualizados

5. **DELETE /categories/:id** - Delete
   - Soft delete da categoria
   - Retorna 200 com confirmação

### Funcionalidades

- **Autenticação**: Todos os endpoints requerem JWT token
- **Validação**: Parsing de body e query params
- **Tratamento de Erros**: `handleUseCaseError` para mapear erros de domínio para HTTP
- **Swagger Annotations**: Documentação completa da API

### Tratamento de Erros

- 400: Bad Request (input inválido)
- 401: Unauthorized (token ausente/inválido)
- 404: Not Found (categoria não existe)
- 500: Internal Server Error (erros inesperados)
