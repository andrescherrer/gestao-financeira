# BUD-001: Criar value object BudgetID

**Data:** 2025-12-27 09:43:47  
**Sprint:** 3.1 - Budget Context - Backend  
**Status:** ✅ Completo

## Resumo

Criação do value object `BudgetID` para representar identificadores únicos de orçamentos no domínio. O value object garante que apenas UUIDs válidos sejam aceitos como identificadores de orçamentos.

## Implementação

### Arquivos Criados

- `backend/internal/budget/domain/valueobjects/budget_id.go`
- `backend/internal/budget/domain/valueobjects/budget_id_test.go`

### Funcionalidades

1. **NewBudgetID**: Cria um novo `BudgetID` a partir de uma string, validando se é um UUID válido
2. **GenerateBudgetID**: Gera um novo `BudgetID` com UUID aleatório
3. **MustBudgetID**: Cria um `BudgetID` e faz panic se inválido (útil para testes)
4. **Value**: Retorna o valor do ID como string
5. **String**: Implementa `fmt.Stringer`
6. **Equals**: Compara dois `BudgetID` para igualdade
7. **IsEmpty**: Verifica se o ID está vazio

### Validações

- ID não pode ser vazio
- ID deve ser um UUID válido (formato UUID v4)

### Testes

Testes unitários cobrindo:
- Criação com UUID válido
- Criação com UUID inválido
- Criação com string vazia
- Geração de IDs únicos
- Métodos de comparação e verificação

## Padrão Seguido

Seguido o mesmo padrão usado em `AccountID`, `TransactionID` e `CategoryID`, garantindo consistência na arquitetura.
